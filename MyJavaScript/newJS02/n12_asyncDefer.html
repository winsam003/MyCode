** async, defer, strict Test **

1. script  의 위치에 따른 효율성
1.1) head 영역에 두는 경우 
	=> html 화면이 준비되기전 JS를 읽기 때문에 JS에서 html의 엘리먼트를 불러올 수 없음
	=> 많은 시간이 소요 됨.

1.2) 바디의 끝부분
	=> 페이지가 준비된 후 실행 JS 받기전에 페이지가 출력됨
	=> 이때 JS 의존적인 콘텐츠는 출력되지 못함

1.3) 해결 (외부문서의 경우 적용) : async , defer
	=> head + async
		- boolean 타입으로 선언하는 것만으로도 true
		- html parsing 도중 -> async 읽으면 병렬로 js파일 fetch & 실행(html은 이때 멈춤) 완료후 
		  -> 다시 html parsing
		- 복수의 js 문서의 경우 정의된 순서와 무관하게 먼저 fetching 된 파일부터 실행 
		  ( js가 순서 의존적이라면 문제가 될수있음 )
	=> head + defer
		- 병렬로 js파일 fetch -> html parsing 마치면 웹 페이지를 먼저 출력 -> js 실행
		- async 의 단점 보완.  
		- 복수의 js 문서의 경우 선언된 순서대로 실행
